Modelo OSI

El modelo OSI nace como una propuesta de la organización Internacional de Normas (ISO) para estandarizar los protocolos utilizados hasta ese momento, este modelo no es una arquitectura de red, ya que carece de la especificación de los servicios y protocolos exactos a utilizar en cada capa, solo se indica lo que se debe de hacer, sin embargo, la ISO también cuenta con estándares para todas las capas aunque no son parte del modelo en sí. En 1995 nace el modelo de Interconexión de Sistemas Abiertos (OSI). El modelo se divide en siete capas que siguen la siguiente lógica:

Debe haber una capa donde se requiera un nivel diferente de abstracción
Cada capa debe realizar únicamente para lo que fue prevista
La función de cada capa debe ser elegida tomando en cuenta los protocolos estandarizados internacionalmente
Los límites de las capas deben minimizar el flujo de información a través de las interfaces
Las capas deben ser autosuficientes para realizar su función y no requerir de la agrupación de funciones adicionales con la finalidad de mantener las capas lo más pequeño posible para evitar que se vuelvan inmanejables.


Capas:
Física: Es la capa donde se transmiten los datos bit por bit a través de medios físicos como cables de ethernet o comunicación inalámbrica como WIFI o Bluetooth. Se mandan señales eléctricas u ondas específicas indicando un 1 o 0.
Enlace de datos: Maneja la información enviada o recibida en “tramas” las cuales son paquetes de datos más pequeños que los enviados a través de la capa de red. Se encarga del control de flujo y manejo de errores para evitar una trama que contenga información errónea y sea enviada o recibida incorrectamente.
Capa de red: Maneja la transferencia de los datos entre los dos dispositivos comunicándose. Si la comunicación es dentro de la misma red, la capa de red no se utiliza. Esta capa maneja la información en unidades llamadas paquetes las cuales son las que se envían a través de la red.
Transporte: Esta capa se encarga de decidir si romper un paquete de la capa superior en uno más pequeño si es necesario, determina el tipo de servicio que debe de tener la capa de sesión y determina los usuarios de la red. Aísla toda la información de las capas superiores para que la información agregada y modificada de las siguientes capas no afecte su integridad.
Sesión: Esta capa las herramientas necearias para establecer conexión entre diferentes máquinas en este caso explicaremos tres de sus tareas
El control de diálogos, el cual se encarga de llevar el control de quienes transmiten información.
El manejo de tokens para evitar que dos partes intenten la misma operación crítica al mismo tiempo.
La sincronización que evita la pérdida de información mediante puntos de referencia a lo largo de las transmisiones para poder reanudar en caso de alguna interrupción
Presentación: Esta capa se enfoca en la sintaxis y semántica de la información transmitida, por lo general se definen estructuras de datos a intercambiar con una codificación estándar, para ello la capa de presentación se encarga del manejo de dichas estructuras y permite definir e intercambiar estructuras más complejas.
Aplicación: En esta capa se establecen los protocolos de comunicación utilizados con mayor frecuencia por los usuarios como por ejemplo el protocolo HTTP utilizado a lo largo de toda la web. Estos protocolos se utilizan para transferir archivos, enviar y recibir correos, esparcir noticias, entre otros.

Modelo TCP/IP

El modelo TCP/IP es un modelo de red que describe cómo se comunican los dispositivos en una red de computadoras. Se divide en cuatro capas: la capa de aplicación, la capa de transporte, la capa de red y la capa de enlace de datos.

Capa de aplicación
Responsable de la comunicación entre aplicaciones

Capa de transporte
Responsable de la transmisión de datos entre dispositivos y asegura que los datos lleguen correctamente

Capa de red
Se encarga de enrutar los datos a través de la red hacia su destino

Capa de enlace de datos
Responsable de la transmisión de datos a través de medios físicos como cables o inalámbricos.

Este modelo es utilizado en la mayoría de las redes de computadoras y es esencial para el funcionamiento de internet. Cada capa tiene su propio conjunto de protocolos y estándares que permiten que los dispositivos en la red se comuniquen y compartan datos de manera efectiva.

Comparación entre OSI y TCP/IP

Un ejemplo de aplicación del modelo TCP/IP sería el envío de un correo electrónico utilizando el protocolo SMTP. En este ejemplo, la capa de aplicación se encargaría de la comunicación entre el cliente de correo electrónico y el servidor de correo electrónico. La capa de transporte utilizaría el protocolo TCP para entregar los datos de manera confiable. La capa de red se encargaría del enrutamiento de paquetes a través de la red utilizando el protocolo IP. Y finalmente, la capa de enlace de datos se encargaría de enviar los paquetes a través de la red utilizando el protocolo Ethernet.
Por otro lado, en el modelo OSI, un ejemplo de aplicación podría ser el uso del protocolo FTP para transferir archivos. En este caso, la capa de aplicación se encargaría de la comunicación entre el cliente FTP y el servidor FTP. La capa de transporte se encargaría de la entrega de datos utilizando el protocolo TCP. La capa de red se encargaría del enrutamiento de paquetes utilizando el protocolo IP, mientras que la capa de enlace de datos se encargaría de enviar los paquetes a través de la red utilizando el protocolo Ethernet. Además, la capa de sesión en el modelo OSI se encargaría de establecer, mantener y finalizar la sesión entre el cliente FTP y el servidor FTP, una función que no está presente en el modelo TCP/IP.
En resumen, aunque ambos modelos sirven para explicar cómo los dispositivos en una red de computadoras se comunican entre sí, el modelo TCP/IP es más utilizado en la actualidad y se enfoca más en la conectividad de redes, mientras que el modelo OSI es más teórico y es utilizado principalmente para propósitos educativos.

Es importante tener en cuenta que la necesidad de utilizar un cable cruzado o recto depende de los estándares de conexión utilizados por los elementos de la red y la forma en que se transmite la señal. En general, si se conecta un dispositivo de red a otro dispositivo de red de la misma categoría (como un switch a un switch), se necesita un cable crossover para cruzar los pares de transmisión y recepción. Si se conecta un dispositivo de red a un dispositivo de red de una categoría superior o inferior (como una computadora a un switch), se necesita un cable recto para mantener los pares de transmisión y recepción en el mismo orden.

PHP

PHP es un lenguaje de programación de código abierto utilizado principalmente para el desarrollo de aplicaciones web y la creación de contenido dinámico en sitios web. Es un lenguaje interpretado, lo que significa que no necesita ser compilado antes de ser ejecutado.

¿Para qué sirve el lenguaje PHP?

PHP se utiliza principalmente para el desarrollo de aplicaciones web, incluyendo el manejo de formularios, la creación de contenido dinámico, la interacción con bases de datos, la gestión de sesiones y cookies, y la creación de sistemas de comercio electrónico y sitios web complejos.

¿Cuál es la diferencia entre los archivos HTML y PHP?

HTML es un lenguaje de marcado utilizado para la creación de páginas web estáticas, mientras que PHP es un lenguaje de programación utilizado para crear páginas web dinámicas. Los archivos HTML son estáticos, lo que significa que no pueden cambiar una vez que se han cargado en el navegador, mientras que los archivos PHP pueden generar contenido dinámico en función de los datos y las condiciones.

¿Para qué sirve la etiqueta ?

La etiqueta se utiliza en PHP para delimitar el código PHP dentro de un archivo HTML o de otro tipo. Todo el código PHP se escribe dentro de estas etiquetas.

¿Cuáles son los pasos y requisitos para cargar un archivo PHP en un servidor?

Para cargar un archivo PHP en un servidor, se necesitan tres cosas: un servidor web que admita PHP (como Apache), un intérprete de PHP (como PHP-FPM) y un archivo PHP. El archivo PHP debe estar guardado en la carpeta raíz del servidor web (generalmente llamada htdocs o public_html). El archivo se puede acceder a través de una URL que apunte al servidor y la ubicación del archivo.

Elabora un programa básico “Hola mundo” con PHP, describe el funcionamiento y elementos utilizados. Inserta el programa embebido con repl.it en tu página.

En este programa se utiliza la función "echo" para imprimir en la pantalla el mensaje "¡Hola mundo!". La etiqueta se utiliza para delimitar el código PHP en el archivo HTML.

¿Para qué sirve la función echo en PHP?

La función "echo" en PHP se utiliza para imprimir contenido en la pantalla del usuario. Puede utilizarse para imprimir texto, variables y otros tipos de datos. La función "echo" también se puede utilizar para imprimir código HTML dentro de una página PHP.

¿Cuál es la sintaxis de la programación en PHP?

La sintaxis básica de PHP es similar a la de otros lenguajes de programación como C, C++ y Java. Las instrucciones se separan con un punto y coma (;) y las estructuras de control de flujo, como los bucles y las declaraciones condicionales, utilizan las llaves {} para delimitar el código.

Describe la diferencia y consideraciones especiales al utilizar caracteres en mayúsculas en PHP, al utilizarlo en palabras reservadas y en variables.

En PHP, se utilizan convenciones de nomenclatura para diferenciar entre palabras reservadas, variables y constantes. Las palabras reservadas en PHP, como "if" y "else", están escritas en minúsculas y no deben utilizarse como nombres de variables o constantes. Las variables y las constantes, por otro lado, se escriben generalmente en minúsculas y se separan con guiones bajos (_).

Además, las constantes en PHP se escriben generalmente en mayúsculas para diferenciarlas de las variables. Es importante tener en cuenta que las constantes son valores que no cambian durante la ejecución del programa, mientras que las variables pueden cambiar su valor.

¿Cómo se declaran variables en PHP?

En PHP, se puede declarar una variable simplemente asignando un valor a un nombre de variable. El nombre de la variable debe comenzar con un signo de dólar ($), seguido del nombre de la variable. Por ejemplo:

$mi_variable = "Hola Mundo";

Cómo se declaran funciones en PHP?

Para declarar una función en PHP, se utiliza la palabra clave "function", seguida del nombre de la función y una lista de parámetros entre paréntesis. La definición de la función se encierra en llaves {}. Por ejemplo:

function suma(n1,n2){ $resultado = n1 + n2; return $resultado; }

¿Cuáles ciclos de programación están disponibles en PHP?

En PHP, están disponibles los ciclos de programación "while", "do-while", "for" y "foreach". Cada uno de estos ciclos se utiliza para ejecutar un bloque de código varias veces, en función de una condición o una lista de elementos.

¿Cuáles funciones de condicional están disponibles en PHP?

En PHP, están disponibles las funciones de condicional "if", "else", "elseif", "switch" y "case". Estas funciones se utilizan para evaluar una expresión o una variable y ejecutar diferentes bloques de código en función del resultado de la evaluación. Por ejemplo:

if ($edad > 18){ echo "Mayor de Edad"; } else{ echo "Menor de Edad"; }

En este ejemplo, se utiliza la función "if" para evaluar la variable $edad y ejecutar el código correspondiente en función de su valor. Si $edad es mayor o igual a 18, se imprime "Eres mayor de edad", de lo contrario se imprime "Eres menor de edad".

Identificación de los diferentes métodos reservados para el tratamiento de archivos:

En PHP, existen diferentes métodos para trabajar con archivos, algunos de los más comunes son los siguientes:

fopen(): Abre un archivo y devuelve un identificador de archivo.

fread(): Lee un número especificado de bytes de un archivo abierto.

fwrite(): Escribe en un archivo abierto.

fclose(): Cierra un archivo abierto.

fgets(): Lee una línea de un archivo abierto.

feof(): Comprueba si el final de un archivo ha sido alcanzado.
